# UI、前端与配置设计笔记

本文记录 Image Agent Studio 在工作台 UI、无限画布、模型配置和生成流程上的设计基准。参考项目只用于学习公开的产品结构和交互思路，不复制代码、截图、品牌资产或素材库。

## 参考方向

- `basketikun/infinite-canvas`：重点学习无限画布中的节点、连线、悬浮工具条、缩放控制和配置节点化思路。
- `codegrazier/cpa-image`：重点学习本地优先的三栏工作台、紧凑设置弹窗、任务状态、并发/间隔控制和测试连接按钮。

## 已确定的产品取向

- 工作站分成两种创作模式：`单次生图` 用于快速出图，`无限画布` 用于多轮分支、对比和衍生。
- 配置面板只放必要字段和动作：接口类型、URL、Key、模型、同步/测试状态。长说明放进文档，不塞进弹窗。
- 模型同步必须可见、可手动触发。用户改 URL 或 Key 后，应能立即点击同步，而不是等待系统“看起来像自动发生了”。
- 画布节点之间使用轻量连线，节点操作靠悬浮工具和右键/快捷动作，不在画布上堆常驻大按钮。
- 灵感库和参考图不应遮挡主工作区。默认以抽屉、侧栏、缩略图和提示词专区承载；没有图片的内容归入提示词专区。
- 队列状态要短、稳、可恢复：`queued`、`running`、`done`、`failed`、`canceled`、`unknown`。用户要能知道任务仍在等、已经失败，还是结果未知。

## 配置面板规则

- 字段按调用链排序：接口类型 -> URL/Key -> 模型 -> 同步状态 -> 操作按钮。
- 所有关键动作都要有显式按钮：同步模型、测试连接、清除、完成。
- 状态文本不超过两行；技术细节留给错误详情或日志。
- 手动密钥默认只存在当前浏览器会话；如果以后提供长期保存，需要明确的开关和提示。
- Provider 适配留在适配层，不把 NewAPI、Sub2API 或其他上游的特定字段泄漏到主 UI。

## 画布与节点规则

- 单张图是节点，不是普通图片堆叠。节点需要保留 prompt、模型、尺寸、来源、父节点、生成时间和失败原因。
- 连线只表达创作关系，线条要细，避免干扰图片预览。
- 节点常用操作放悬浮工具条：继续生成、设为参考图、查看提示词、下载、删除。
- 大图只在用户打开预览时加载；列表和灵感广场优先使用缩略图。
- 刷新后应恢复会话、画布节点、队列状态和历史图片引用。

## 后续优化清单

- 增加“测试连接”按钮：检测 URL、Key、`/v1/models`、图片生成路由和编辑路由是否可用。
- 把队列设置做成紧凑控件：并发数、间隔、取消、清理失败、清理完成。
- 给无限画布补节点悬浮工具条和更轻的连线样式。
- 给灵感库补“提示词专区”和缩略图缓存状态，避免破图拖慢首屏。
- 把设置、灵感库、历史图库继续懒加载，首屏只保留工作台核心。
